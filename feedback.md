## 📊 [Part 1] 구현물 평가

### 1) PID 트래킹 및 세이프가드
*   **평가**: 설계도(`plan.md`)에 설정된 튜닝 게인($K_p=0.15$, $K_i=0.01$, $K_d=0.05$) 및 화면 중심(320, 240) 기준 오차 계산 공식이 정확하게 이식되었습니다.
*   **세이프가드**: 스무딩 처리 후 조준 출력을 ±80px 한계로 클램핑하여 오버슈트로 인한 급발진을 물리적으로 방어하는 연동 구조가 안전하게 기동됩니다.
*   **안전 규정 준수**: 핵심 하드웨어 제어 모듈(`motor_esp32.py`)을 수정하지 않고, 비전 엔드포인트 단에서 PID 가상 필터를 적용한 후 `state.point` 좌표로 주입하는 우회 안전 연동 규정을 완전하게 준수하였습니다.

### 2) Anti-Windup 및 인증 보안
*   **Anti-Windup**: 적분 오차가 무한히 발산하여 제어량이 튀는 현상을 막기 위해, 누적 최대치를 `100.0`으로 제동하는 포화 방지(Anti-Windup) 장치가 정상 작동합니다.
*   **Auth 보안**: 비밀번호를 단방향 PBKDF2 해싱 암화화하여 저장하며, 로그인 검증 시 실패 원인(이메일 부재 / 비밀번호 불일치)에 대해 동일한 401 에러 메시지를 반환함으로써 공격자의 가입 계정 역탐색 시도를 효과적으로 방어합니다.

## 🔍 [Part 2] 잠재적 위협 & 취약점 분석 및 조치 완료

### 1) 리셋(gap) 직후 첫 호출 시 dt 미초기화로 인한 즉시 Windup 포화 버그
*   **지적**: 최초 호출 시 $dt$가 초기화되지 않고 유닉스 타임스탬프 또는 타겟 소실 시간만큼 계산되어 첫 프레임에서 적분값 `state._integral_x`가 즉시 한계치(±100)로 튀어버려 적분 제어가 무력화되는 버그가 있었습니다.
*   **조치 완료 (All Green)**: `gap` 발생 또는 초기화 분기 발생 시 $dt = 0.001$로 즉시 리셋되도록 보강하여 리셋 직후 적분 폭주 위험을 원천 방지하였습니다. (검수기 최종 확인 완료)

### 2) 미분(D) 댐핑 테스트 시나리오 검증력 강도 부족
*   **지적**: 기존 D항 검증 테스트가 단순히 오차가 감소했는지만 확인하여 미분(D)의 감속 브레이크 동작에 따른 `output_x` 연산 억제력이 작동했는지 확인하지 못했습니다.
*   **조치 완료 (All Green)**: `test_pid.py` 내의 댐핑 테스트에서 미분 제동력(-50.0)이 비례 제어량(+1.5)보다 압도적으로 강해 실제 조준점 `state._smooth_tx`가 목표치보다 급격히 감속되었는지에 대한 `self.assertTrue(state._smooth_tx < 390.0)` 검증식을 추가하여 테스트 신뢰도를 보강 완료하였습니다. (검수기 최종 확인 완료)

## ⚡ [Part 3] 라즈베리파이 최적화 제안

라즈베리파이 4 임베디드 리소스의 극소화 및 저지연(Low Latency) 구동을 위한 튜닝 권장안입니다.

1. **MediaPipe 경량화 스펙 고수**:
   * CPU 연산 최소화를 위해 `max_num_hands=1` 및 근거리 경량 모델 `model_selection=0` (2m 내 최적화) 설정을 유지하십시오.
2. **비디오 스트림 GIL 병합 억제**:
   * Flask 스트리밍(`gen_frames`) 시 최대 프레임 출력을 20 FPS로 한계 제한하고, JPEG 인코딩 퀄리티를 `50`으로 조율하여 전송 지연으로 인한 파이썬 GIL(Global Interpreter Lock) 락 현상을 예방합니다.
3. **시리얼 전송 대역폭 최적화**:
   * `POS` 상태 질의 전송 주기를 Auto 트래킹 모드 시에는 60ms, Manual 수동 조작 모드 시에는 500ms로 조절하여 UART 대역폭 낭비를 억제하십시오.
