# 하드웨어 안전 규정 (Hardware Safety Rules)

AI Vision Tracker의 하드웨어는 물리적인 충돌 시 모터와 기어박스에 영구적인 손상이 발생할 수 있으므로, 소프트웨어 수정 시 다음 규칙을 반드시 준수해야 합니다.

1. **M1(수평) 및 M2(수직) 물리적 리밋 확인**
   - M1은 기본 `±180도`(360도)의 회전 범위를 가집니다.
   - M2는 위로 `+45도`, 아래로 `-45도`의 회전 범위를 가집니다.
   - 이 범위를 초과하는 명령을 절대 ESP32 펌웨어로 전송해서는 안 됩니다.

2. **2D Safe Zone 적용 원칙**
   - 카메라가 아래를 향할 때(-25도 미만, 즉 -45~-25 구간), M1의 회전은 `±80도`(160도)로 제한되어야 합니다. (기둥 충돌 방지)
   - 반대로 M1이 ±80도 밖에 있으면 M2는 -25도 아래로 내려갈 수 없습니다.
   - `web/routes/core.py`의 `joystick_dir`에서 모터 좌표계가 이 규칙을 벗어날 가능성이 있는지 항상 테스트해야 합니다.

3. **자동 감속 (Soft Braking) 필수 유지**
   - 조이스틱 등 외부 입력에 의한 이동 시, 목적지 도달 전 15~20도 부근에서 반드시 점진적으로 감속하도록 로직이 작성되어야 오버슛을 방지할 수 있습니다.

4. **복귀 보장 (Escape Guarantee) 필수 유지**
   - 리밋 차단은 반드시 "범위를 벗어나는 방향"에만 적용해야 합니다.
   - 범위 안으로 되돌아오는 방향의 이동은 어떤 경우에도 차단하거나 0으로 감속해서는 안 됩니다. (리밋이나 오버슛 상태에서 갇히는 것을 방지)
   - 사용자 설정 리밋 병합 결과 범위가 뒤집히면(min >= max) 사용자 리밋을 무시하고 하드웨어 한계로 복귀해야 합니다.
