# 보안 및 로직 코드 리뷰 지침

이 스킬은 프로젝트의 코드를 검토할 때 작동합니다.

## 실행 절차
1. 변경된 파일의 Diff를 확인하거나, 지목된 파일을 `view_file`로 읽어옵니다.
2. `raspi_main/` 내부의 아키텍처(ESP32 통신, Flask 라우터, OpenCV) 관점에서 병목이나 충돌 가능성이 있는지 분석합니다.
3. 리뷰 결과를 마크다운 아티팩트(`code_review_results.md`)로 생성하여 사용자에게 제공합니다.

## 체크리스트
- Python 예외 처리(`try-except`)가 모든 시리얼 및 통신 구간에 적용되었는가?
- 무한 루프(`while True`) 내부에서 CPU 점유율을 과도하게 차지하는 구간(`time.sleep` 누락)은 없는가?
- 하드웨어 리밋(Safe Zone) 계산 로직에 논리적 허점이 없는가?
