# AI Vision Tracker - Workspace Rules

이 파일은 AI Vision Tracker 프로젝트에 대해 저(Gemini 기반 AI 에이전트)에게 지시할 프로젝트 단위의 **규칙과 가이드라인**을 모아두는 곳입니다.
글로벌 규칙(Global Rules) 외에, 이 프로젝트만을 위해 특별히 지켜야 할 사항들을 작성해 주세요.

## 🛠️ 프로젝트 아키텍처 규칙
- **웹 서버 프레임워크**: Python Flask (포트 5000)
- **통신 방식**: 비동기 직렬(Serial/UART) 통신으로 ESP32/Arduino와 연결
- **모터 제어 로직 (Python)**: ESP32 펌웨어를 수정하지 않고 `routes/core.py` 등 파이썬 단에서 소프트웨어 리밋(안전 구역, 자동 제동)을 전적으로 처리해야 합니다.
- **UI/UX 디자인**: Apple 스타일의 미니멀리즘 플랫 디자인을 유지합니다. (Glassmorphism, 둥근 모서리, 단순하고 직관적인 인터페이스)

## ⚠️ 커밋 및 브랜치 전략
- 배포용 안정화 버전은 `main` 브랜치에 저장합니다.
- 새로운 기능 테스트 및 수정 사항은 `dev` 브랜치에서 진행 후 병합합니다.

## 🔄 코드 수정 가이드
- 하드웨어 안전 직결 코드(모터 리밋 등)를 수정할 때는 관성 밀림, 2D 구조물 충돌 위험을 먼저 계산하세요.
- AI 트래킹 관련(`detector.py`)을 수정할 땐, 추적 성능 저하 여부를 면밀히 검토하고 복잡도를 낮추는 쪽을 지향하세요.
