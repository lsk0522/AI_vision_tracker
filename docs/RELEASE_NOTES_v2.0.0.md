# 🏆 AI Vision Tracker v2.0.0 Release Notes

이번 v2.0.0 릴리즈는 **프로젝트 구조 전면 재정비와 안전 리밋 시스템 재설계**를 담은 메이저 업데이트입니다. 흩어져 있던 백엔드 파일을 역할별 패키지로 정리하고, 2,700줄짜리 프론트엔드 스크립트를 17개 모듈로 분리했으며, 리밋에 걸리면 빠져나올 수 없던 고질적인 버그를 복귀 보장(Escape Guarantee) 설계로 해결했습니다.

---

## ✨ 주요 업데이트 내용

### 1. 📦 프로젝트 구조 전면 재정비
평면적으로 흩어져 있던 파이썬 파일들을 역할별 패키지로 재구성하여 유지보수성을 크게 높였습니다.

```text
config/    전역 상태 (state.py) — 모터 좌표 · 리밋 · 펌웨어 버전
core/      영상 처리 — detector(YOLO) · camera · capture · cli_ui · logger
hardware/  하드웨어 통신 — motor_esp32 · motor_arduino · serial_utils
web/       웹 대시보드 — routes(API) · static/js · templates
data/      데이터 — YOLO 모델 · 학습 데이터 · 캡처 이미지
scripts/   설치 · 실행 · 원격 트래킹 스크립트
tests/     테스트 스크립트
```

### 2. 🧩 대시보드 프론트엔드 ES 모듈 분리
한 파일에 몰려 있던 2,716줄의 `script.js`를 기능별 **17개 ES 모듈**(`web/static/js/`)로 분리했습니다.
- 공유 상태(`state.js`)를 중심으로 조이스틱, 크로스헤어 렌더링, 모터 상태/설정, 카메라 설정, 갤러리, 펌웨어 업로드 등이 각각 독립 모듈로 동작합니다.
- 이 과정에서 **숨어 있던 치명적 버그**를 발견·수정했습니다: 존재하지 않는 UI 요소를 참조하는 코드 한 줄이 예외를 던져, 그 뒤의 모든 초기화(갤러리·모터 상태창·카메라 설정·펌웨어 업로드·ESC 키 등)가 실행되지 않고 있었습니다.

### 3. 🛡️ 소프트웨어 리밋 재설계 — 복귀 보장 (Escape Guarantee)
리밋(또는 관성 오버슛)에 걸린 뒤 **반대 방향으로도 움직이지 않던 버그**를 근본적으로 해결했습니다.
- **원인**: 사용자 설정 리밋 병합 시 부호 처리 오류로 리밋 범위 자체가 붕괴(min ≥ max)되면 양방향이 동시에 차단될 수 있었습니다.
- **해결**: 리밋을 *벗어나는* 방향만 차단하고, 범위 안으로 *되돌아오는* 방향은 어떤 경우에도 차단·감속하지 않도록 재설계했습니다. 병합 결과가 붕괴되면 사용자 리밋을 무시하고 하드웨어 한계로 자동 복귀합니다.
- 감속 구간에 **최소 서행 속도**를 보장하여, 리밋 직전에 모터가 멈춘 것처럼 보이던 현상도 함께 해결했습니다.
- 18가지 시나리오 시뮬레이션(리밋 도달 → 반대 방향 복귀, 오버슛 상태 탈출, 2D 결합 구석 탈출 등)을 전부 통과했습니다.

### 4. 📏 새 작동 범위 적용
| 축 | 작동 범위 | 비고 |
|:---:|:---|:---|
| **M2 (수직)** | 아래 **-45°** ~ 위 **+45°** | 기존 +50/-40 → 대칭 ±45로 변경 |
| **M1 (수평)** | 기본 **360°** (±180°) | 카메라가 **-25° 아래**를 볼 때 **160°** (±80°)로 자동 제한 |

기존의 2D 충돌 방지 결합(수평이 ±80° 밖이면 수직 -25° 이하 차단)은 그대로 유지됩니다.

### 5. 🤖 추적 엔진 YOLO 전환 문서화 및 정확도 수정
- README와 문서를 실제 코드에 맞게 **YOLO INT8(TFLite) 파이프라인** 기준으로 전면 갱신했습니다 (기존 CSRT 문서는 코드와 불일치).
- 자동 추적의 화면 중심 좌표가 `320×240`으로 하드코딩되어 있어 해상도 변경 시 조준이 어긋나던 문제를 실제 프레임 크기 기준으로 수정했습니다.

### 6. 🔒 보안 수정
- 갤러리 삭제 API(`/delete/<filename>`)의 **경로 탐색(Path Traversal) 취약점**을 수정했습니다. 조작된 파일명으로 저장 폴더 밖의 파일을 삭제할 수 있던 문제로, `werkzeug.utils.safe_join` 검증을 추가했습니다.

### 7. 🌐 랜딩 페이지 개편 (website/)
- 히어로 3D 패널 다크 테마 통일, 실측 스펙 스트립, 실물 CAD 도면 삽입, 기술 스택 마퀴, 푸터 보강.
- 시뮬레이션 데모에서 개발자용 축 튜닝 패널을 숨기고(`?debug=1`로만 접근) 방문자용 실시간 각도 표시를 추가했습니다.
- 발표를 위해 PAN/TILT·CSRT·PID 같은 전문 용어를 **1축(수평)/2축(수직)** 등 쉬운 표현으로 순화했습니다.
