# 📖 AI Vision Tracker 사용자 가이드 (USER GUIDE)

이 문서는 **AI Vision Tracker (Turret)** 프로젝트를 처음 접하는 분들이 하드웨어를 조립하고, 소프트웨어를 설치하여, 성공적으로 웹 UI를 통해 객체를 추적할 수 있도록 돕는 상세 매뉴얼입니다.

---

## 1. 하드웨어 결선 가이드 (Hardware Setup)

본 프로젝트는 AI 연산을 담당하는 **라즈베리 파이(또는 PC)** 와 모터를 실시간 제어하는 **ESP32** 가 협력하는 구조입니다.

### 📌 부품 준비물
- 라즈베리 파이 4 (또는 일반 윈도우/맥 PC)
- ESP32 보드 (추천: Lolin D32 등 FreeRTOS 듀얼코어 지원 보드)
- DM542 스테퍼 모터 드라이버 (2개: 수평 Pan, 수직 Tilt)
- NEMA-17 스텝 모터 (2개) 및 전원 공급 장치 (SMPS 12V/24V)
- USB 웹캠

### 🔌 배선 다이어그램 (ESP32 ↔ DM542)
ESP32와 DM542 드라이버를 다음과 같이 연결합니다.

| ESP32 핀 | DM542 핀 | 역할 |
|:---|:---|:---|
| **GPIO 18** | `PUL+` (Pan) | 수평 방향 회전 펄스 |
| **GPIO 19** | `DIR+` (Pan) | 수평 방향 회전 방향 |
| **GPIO 22** | `PUL+` (Tilt)| 수직 방향 회전 펄스 |
| **GPIO 23** | `DIR+` (Tilt)| 수직 방향 회전 방향 |
| **GND** | `PUL-`, `DIR-` | 공통 접지 (Common Ground) |

> **주의:** 모터 드라이버의 전원(`VCC`, `GND`)은 반드시 별도의 강력한 SMPS 외부 전원을 사용해야 합니다. 절대 ESP32의 전원을 모터 드라이버 메인 전원으로 사용하지 마세요.

## 2. 소프트웨어 설치 (Software Setup)

### 1) Python 환경 세팅
AI 트래커와 웹 서버를 돌리기 위해 Python 3.11 이상의 환경이 필요합니다. 터미널을 열고 아래 명령어를 입력하여 필요한 라이브러리를 설치합니다.

```bash
# 필수 라이브러리 설치
pip install flask opencv-python opencv-contrib-python numpy pyserial
```
> `opencv-contrib-python`은 강력한 객체 추적 알고리즘인 **CSRT Tracker**를 사용하기 위해 반드시 필요합니다.

### 2) Arduino CLI 설치 (선택사항)
웹 UI에서 **FOTA (원격 펌웨어 업데이트)** 기능을 사용하려면 PC나 라즈베리 파이에 `arduino-cli`가 설치되어 있어야 합니다. (수동으로 아두이노 IDE를 통해 업로드할 경우 생략해도 됩니다.)

---

## 3. 실행 및 웹 UI 접속

1. 하드웨어(ESP32, 카메라)가 모두 PC/라즈베리파이에 USB로 연결되어 있는지 확인합니다.
2. 터미널에서 아래 명령어로 메인 서버를 실행합니다.
   ```bash
   python main.py
   ```
3. 크롬이나 사파리 등 모던 웹 브라우저를 열고 다음 주소로 접속합니다.
   - 로컬에서 실행 시: `http://localhost:5000`
   - 스마트폰이나 다른 PC에서 접속 시: `http://<실행한_PC의_IP주소>:5000`

## 4. 웹 인터페이스 조작법

화면에 접속하면 Apple 스타일의 예쁜 반투명(Glassmorphism) UI가 여러분을 반깁니다.

### 🎯 물체 추적 (Tracking) 시작하기
1. **모드 전환**: 오른쪽 컨트롤 패널에서 <b>'물건 학습하기'</b> 버튼을 클릭합니다.
2. **대상 지정**: 카메라 화면 위에서 추적하고 싶은 물체 주변을 **마우스(또는 터치)로 드래그**하여 파란색 박스(ROI)를 그립니다.
3. **학습 완료**: 박스를 다 그리면 자동으로 **'LOCKED'** 상태가 되며 터렛이 해당 물체를 따라가기 시작합니다!
4. **탈출**: 언제든지 키보드의 `ESC` 키를 누르면 추적이 초기화되고 대기 상태로 안전하게 복귀합니다.

### 🕹️ 수동 제어 (Joystick)
- 왼쪽 아래에 있는 **가상 조이스틱**을 마우스로 드래그하거나 스마트폰으로 터치하여 터렛을 수동으로 움직일 수 있습니다. 

### ⚙️ 모터 튜닝 (Motor Tuning)
- **최대 속도 (Hz)**: 모터가 낼 수 있는 최고 속도입니다. (권장: `3000 Hz`)
- **가속도 (Hz/ms)**: 출발하거나 정지할 때 얼마나 부드럽게 가속할지 결정합니다. 수치가 높을수록 팍! 튀어나갑니다. (권장: `8.0 Hz/ms`)
- **px당 스텝 이동량**: 화면상에서 물체가 1픽셀 이동했을 때 모터가 몇 스텝 돌아가야 하는지(기어비 반영) 설정합니다.

## 5. FOTA (원격 펌웨어 업데이트) 기능

만약 서버 버전과 ESP32의 코드가 맞지 않으면 화면에 <b>'펌웨어 불일치 경고'</b> 모달이 뜹니다.
이때 당황하지 마시고 화면에 있는 **"최신 펌웨어 업로드"** 버튼을 누르세요! 
내부적으로 `arduino-cli`를 사용해 최신 C++ 코드가 자동으로 ESP32로 전송 및 컴파일됩니다.

## 6. 트러블슈팅 (FAQ)

**Q. 카메라 화면이 안 나와요!**
- 웹캠이 다른 프로그램(예: 줌, 디스코드)에서 사용 중인지 확인하세요. `main.py` 실행 전에 다른 카메라 앱을 모두 닫아야 합니다.

**Q. 모터가 "삐~" 소리만 나고 안 돌아가요.**
- 1. 모터 드라이버의 전원선 연결 상태를 확인하세요. 
- 2. 최대 속도(Hz)가 모터가 감당할 수 있는 수치 이상으로 너무 높게 설정되어 있을 수 있습니다. 웹 설정에서 1000Hz 정도로 낮춰보세요.

**Q. 추적(Tracking) 중에 빨간색 테두리가 뜨며 대상을 놓쳐요.**
- **Competitive Tracking** 기능이 발동하여 화면을 벗어난 타겟을 다시 찾고 있는 상태(SEARCHING)입니다. 
- 물체의 이동 속도가 너무 빠르거나 조명이 급격히 바뀌었을 수 있습니다. 조준점 반응 속도를 조절하거나 대상을 다시 학습시켜 주세요.
